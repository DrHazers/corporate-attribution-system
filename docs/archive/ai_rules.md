# AI Coding Rules

## 1. 总体要求
本项目是一个毕业设计项目，目标是构建一个面向产业研究的企业国别归属与业务结构征订系统。AI 在生成代码时，必须优先保证代码清晰、可运行、可维护，不要追求过度复杂设计。

## 2. 技术规则
- 后端使用 Python + FastAPI。
- ORM 使用 SQLAlchemy。
- 开发阶段数据库可使用 SQLite，后续可迁移到 PostgreSQL。
- 图分析使用 NetworkX。
- 默认不引入 Redis、Celery、消息队列、图数据库等复杂依赖。
- 如果需要新增第三方依赖，必须先说明理由。

## 3. 架构规则
- 先实现后端主干，再逐步扩展前端。
- 每次只实现一个小功能，避免一次生成过多代码。
- 优先保证目录结构清晰。
- 新代码必须与当前项目结构一致。
- 不要擅自重构整个项目。

## 4. 业务规则
- 企业的注册地、上市地、实际控制地必须分开存储。
- 实际控制地判定应优先依据最终控制主体。
- 特殊结构如 VIE、红筹、跨国控股需要保留类型标记。
- 控制关系应保留可追溯路径。
- 业务线需要区分主营业务和新兴业务。
- 所有人工征订都必须写入日志。

## 5. 代码风格要求
- 代码要有清晰的中文注释。
- 函数命名和变量命名应直观明确。
- 接口需要定义请求和响应 schema。
- 不要生成过长的单文件代码。
- 新增功能时尽量提供最小可运行示例。

## 6. 与 AI 协作方式
- 在生成代码前，先说明计划修改哪些文件。
- 在修改现有代码前，先阅读相关文件内容。
- 优先给出小步迭代方案。
- 如果需求不明确，优先基于当前项目目标做保守实现，不要擅自扩展范围。

## 7. 环境与依赖规则（重要）

本项目使用 Python venv 虚拟环境，路径为：

D:\graduation_project\corp_attribution_system\venv

所有命令必须基于该虚拟环境执行，禁止使用系统默认环境。

### 1. Python 与 pip

- 所有依赖安装必须使用：
  python -m pip install <package>

- 禁止使用：
  pip install <package>

- 所有 Python 命令默认使用当前 venv 的 python：
  D:\graduation_project\corp_attribution_system\venv\Scripts\python.exe

---

### 2. 服务启动

- 启动 FastAPI 服务必须使用：
  python -m uvicorn backend.main:app --reload

- 禁止使用：
  uvicorn backend.main:app --reload

---

### 3. 测试命令

- 如果需要运行测试，必须使用：
  python -m pytest

- 禁止使用：
  pytest

---

### 4. 未来可能使用的工具（预防性约束）

如果涉及以下工具，必须使用 python -m 方式：

- python -m alembic
- python -m uvicorn
- python -m pytest

---

### 5. 禁止行为

- 不要假设系统 Python 是正确的
- 不要使用全局 pip / uvicorn / pytest
- 不要依赖 PATH 中的默认命令
- 不要执行任何未说明来源的系统级命令

---

### 6. 执行前检查（重要）

在生成涉及命令的方案时，必须确保：

- 命令基于当前项目 venv
- 命令在 Windows PowerShell 下可执行
- 路径与当前项目结构一致

如果不确定，请优先使用：

python -m <module>

---

## 8. 数据库文件与初始化规则（重要）

### 1. 固定数据库文件

- 开发环境数据库固定使用：
  `company.db`

- 测试环境数据库固定使用：
  `test.db`

- 不要使用时间戳、随机数或临时命名生成数据库文件，例如：
  `test_xxx_123.db`
  `test_20260318.db`
  `temp.db`

---

### 2. 测试数据库规则

- 所有测试代码必须复用同一个 `test.db`
- 测试开始前如果 `test.db` 已存在，可以删除后重建，或者清空数据
- 不允许每个测试文件、每次运行、每个测试用例创建不同名字的 `.db` 文件

---

### 3. 初始化规则

- 数据库初始化逻辑必须显式可控
- 不要在模块导入时自动执行建表逻辑
- 建表应通过明确的初始化函数或应用启动阶段执行

---

### 4. 开发约束

- 不允许在代码中自动生成多个 `.db` 文件
- 新增测试或新增模块时，必须继续遵守：
  开发库使用 `company.db`
  测试库使用 `test.db`
